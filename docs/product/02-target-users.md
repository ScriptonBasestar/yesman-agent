# 🎯 Target Users & Use Cases

## 👤 주요 사용자 그룹
| 사용자 | 필요/페인포인트 | Yesman Agent가 주는 가치 |
| --- | --- | --- |
| 백엔드·플랫폼 엔지니어 | 반복적인 tmux/CLI 설정, 다중 서비스 부트스트랩 부담 | 템플릿 기반 세션 자동화, CLI/Workflow 재사용 |
| DevOps / SRE | 배포·장애 대응 시 실시간 상태 공유 필요 | FastAPI + Dashboard로 상태 스트리밍 및 협업 |
| AI 오퍼레이터 / Prompt 엔지니어 | 모델별 실행 정책/맥락 관리 필요 | Agent Runtime에서 Prompt Chain/Context Loader 표준화 |
| PM / Technical Writer | 문서 요약·리뷰 자동화 요구 | 플러그인 + 워크플로 조합으로 문서 처리 파이프라인 구성 |
| 내부 개발자 포털 팀 | 조직 공통 자동화 인터페이스 필요 | CLI/API/Dashboard를 단일 설정으로 배포 가능 |

## 💡 대표 시나리오
1. **코드 리포지터리 리포트 자동화**: 에이전트가 리포지토리를 스캔하고 요약/이슈 목록을 생성, 결과를 대시보드에서 공유.
2. **DevOps 런북 실행**: YAML Workflow로 장애 대응 절차를 정의하고, CLI·API에서 동일한 스크립트를 반복 실행.
3. **문서/PR 리뷰 지원**: LLM 플러그인이 텍스트를 분석하고 코멘트를 생성, 결과를 FastAPI 웹훅 또는 Slack으로 전달.
4. **로컬 개발 환경 부트스트랩**: 프로젝트별 tmux 템플릿과 의존성 체크 스크립트를 자동 실행해 신규 팀원의 온보딩을 단축.
5. **멀티 에이전트 실험**: 서로 다른 LLM Provider를 연결한 워크플로를 구성해 성능 비교 및 A/B 실험을 자동화.
